## REST API 很好、很強 ## 但還是有一些缺點
+ 參數、回傳值型別不固定 + 多版本時，前後端常不匹配 + 會拿多餘的欄位 + 不容易處理巢狀資源 + 會不斷成長的 Endpoint 數量 [參考資料：http://ithel...188294](http://ithelp.ithome.com.tw/articles/10188294)
## 總而言之 ## GraphQL是一個 ## 很新、對前端友善的 API
